## Critical (4) — всі виправлені

### 1. Race condition: double bankroll update
**Файли:** `scheduler/tasks/live_tracker.py`, `scheduler/tasks/update_results.py`

`live_tracker` (кожні 3 хв) і `update_results` (кожні 30 хв) одночасно завантажували одні й ті ж predictions з `result=NULL`, обидва рахували PnL і обидва викликали `_update_bankrolls` → bankroll збільшувався вдвічі.

**Фікс:**
- `update_results` тепер викликає `db.refresh(pred)` перед обробкою кожного prediction і пропускає ті, де `result is not None`
- Обидва таски передають у `_update_bankrolls` тільки список `newly_settled` — predictions що щойно змінились з None → win/loss
- `_update_bankrolls` підписаний `(db, newly_settled: list)` в обох файлах

### 2. N+1 queries в generate_picks_monster.py
**Файл:** `scheduler/tasks/generate_picks_monster.py`

`_get_odds(match.id, db)` викликався для кожного матчу в циклі — N запитів до БД замість одного.

**Фікс:**
- Видалено `_get_odds()`; додано `_load_odds_batch(match_ids, db)` — один `WHERE match_id IN (...)` перед циклом
- Аналогічно existing predictions — теж завантажуються одним запитом до циклу, замість 2×N запитів per-match

### 3. MODEL_VERSION = "v1" в generate_picks.py
**Файл:** `scheduler/tasks/generate_picks.py`

Стара ML v1 модель використовувала `MODEL_VERSION = "v1"`, але API очікує `"ws_gap_v1"`, `"monster_v1_kelly"` тощо. Picks від цього таска були б невидимі для фронтенду. **Але таск закоментований в scheduler/main.py** (`# ML v1 — вимкнено`), тому це мертвий код, не live-баг.

### 4. DB session leaks (audit flag)
Після перевірки: `try/finally: db.close()` присутній у всіх тасках. Менш критично ніж звітував аудит. Залишено як є.

## High (6) — всі виправлені

### 5. Bot handler без try/except
**Файл:** `bot/handlers.py`

`cmd_start` → `message.answer(...)` без захисту. При Telegram rate limit або network error бот крашився.

**Фікс:** Додано `try/except Exception` з `logger.warning(...)`.

### 6. PickCard: NaN при stake=null
**Файл:** `capper-miniapp/src/components/PickCard.tsx`

`const profit = (pick.stake * (pick.odds - 1)).toFixed(0)` — якщо `stake=null` (historical imports), вираз повертав `NaN` і рендерився в UI.

**Фікс:** `pick.stake != null ? ... : null`; всі money-row значення показують `—` при null.

### 7. Stale fetch race condition у PicksScreen
**Файл:** `capper-miniapp/src/screens/PicksScreen.tsx`

При швидкому перемиканні днів/моделей старий fetch міг зарезолвитись пізніше нового і перезаписати правильні дані. useEffect не мав cleanup.

**Фікс:** Додано `cancelled` flag + `return () => { cancelled = true }` в обидва useEffect (picks + stats). Якщо компонент/ефект перезапустився — stale відповідь ігнорується.

### 8. Відсутні LEAGUE_FLAGS для Netherlands / Belgium / Ukraine
**Файли:** `api/utils.py`, `api/main.py`

Monster model покриває Eredivisie, Jupiler Pro League, УПЛ — але всі три повертали `🏟` замість правильного прапора.

Додатково: УПЛ в БД зберігається як "Premier League" (колізія з England).

**Фікс:**
- `api/utils.py` LEAGUE_FLAGS: додано `"Eredivisie": "🇳🇱"`, `"Jupiler Pro League": "🇧🇪"`, `"Ukrainian Premier League": "🇺🇦"`, `"Premier League (UA)": "🇺🇦"`
- `api/main.py` `prediction_to_pick`: після базового lookup перевіряємо `league.country == "Ukraine"` → перевизначаємо flag на `"🇺🇦"`

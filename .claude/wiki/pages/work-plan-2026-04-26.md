## Strategy

**Не міняємо провайдера** — гібрид:
- **Власний Glicko** (Python implementation з match history) → 0% залежність від SStats `/glicko/`

**Всі long-running tasks → VPS Docker scheduler** (незалежно від Mac).

## Blocks

### 1. Self-computed Glicko (~2h) ⭐ CRITICAL — IN PROGRESS
- Glicko-2 algorithm: rating, RD (rating deviation), volatility per team
- Iterative computation з усіх finished matches in DB chronologically
- Snapshot per (match_id, team_id, side) у нову таблицю `team_ratings` AND/OR кеш у redis-like dict
- Replaces SStats `/Games/glicko/{fixture_id}` dependency
- Cron task: оновлення після кожного нового результату
- Works offline, незалежний від мережевих збоїв

### 2. Live opening odds collector (~2h)
- Scheduler task: кожні 30 хв fetch odds для upcoming matches (24h до старту)
- Save з `is_opening=true` (перший observed) + `is_closing=true` (final pre-match)
- Track `line_drift = closing - opening`
- Через 3 місяці накопичення → retrain Gem v3 з market features (`open_prob`, `drift`, `drift_velocity`)
- **Очікуваний impact після retrain:** +5-7pp WR, +5-10pp ROI

### 3. Injuries collector + xG-weighted feature (~2h)
- API-Football `/injuries` (daily)
- + `/players/topscorers` для xG_contribution per player
- Compute per-match: `missing_xg_share` = % атаки команди що відсутня
- Замінює binary `home_has_any_injuries` (mean SHAP 0.0024 — фактично шум)
- **Очікуваний impact:** +2-3pp WR

### 4. Lineups collector + перенос pick generation (~1h)
- API-Football `/fixtures/lineups` (за 1.5h до kick-off)
- Compute `lineup_glicko_strength` = сума Glicko стартового складу
- Detect rotation matches (cup midweek → low-strength lineup)
- **Перенос pick generation:** з 5h до 1.5h перед матчем → даємо час підгрузити lineups
- **Очікуваний impact:** +1-2pp WR

### 5. Deploy Gem v2 на прод (~1h)
- `scheduler/tasks/generate_picks_gem.py`
- API: register `gem_v1` в MODEL_VERSIONS
- Mini-app: 5-та model tab (Gem)
- Backfill історичних Gem picks
- Друга жива модель (Pure + Gem) → диверсифікація

### 6. Pure monitoring dashboard (опціонально)
- Daily report per niche: live n, WR, ROI, ↑/↓ vs historical p_is
- Auto-disable niches з WR<40% на n>=10
- Telegram-style summary в mini-app
